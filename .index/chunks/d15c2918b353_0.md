# Indexing & Context

Agentic Assistants supports **two complementary “index” concepts**:

1. **Vector indexing/search** (runtime feature): chunk files, store embeddings in a vector DB, search via CLI/server.
2. **File-based `.index/` context packs** (developer enablement): model-agnostic, verbose artifacts designed for small-context local LLMs.

## Vector indexing & search

### CLI

```bash
agentic index ./src --collection my-project
agentic search "where is websocket implemented?" --collection my-project -k 10
```

### Server

The server exposes legacy endpoints:

- `POST /index`
- `POST /search`
- `GET /collections`

## `.index/` (model-agnostic) index

The `.index/` folder is **not** a vector store. It’s a file-based knowledge package that can be shipped to (or mounted into) small-context local LLM workflows.

It is designed to be:

- **Model-agnostic**: plain Markdown + JSON
- **Detailed**: includes surfaces (CLI, REST, UI routes), symbols, and chunked file content
- **Stable**: versioned schema so tools can rely on it

### ContextLoader + `agentic context`

The framework includes a context loader that reads `.index/context/*` to produce small “context packs”:

- `.index/context/architecture.md`
- `.index/context/api-surface.md`
- `.index/context/patterns.md`
- `.index/context/ollama-assistant.md`

CLI helpers:

```bash
agentic context summary
agentic context show --task understand
agentic context show --task add_cli_command --copy
```
